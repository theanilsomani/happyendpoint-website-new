# Deploying Happy Endpoint

This site is a static Astro build that deploys to **Cloudflare Pages**.

## Prerequisites

- Node 22.12+
- pnpm 9+
- A Cloudflare account with Pages access
- Git access to this repository

## Local development

```bash
pnpm install
pnpm dev
```

Dev server runs at `http://localhost:4321`.

## Production build

```bash
pnpm build
```

Build output goes to `dist/`. This includes:

- All pre-rendered HTML pages
- `_headers` — security headers (X-Frame-Options, HSTS, Referrer-Policy, cache rules for fonts/assets)
- `_redirects` — combined static legacy-URL redirects + per-slug free-dataset redirects (generated from content at build time)
- `pagefind/` — static search index (built by the `postbuild` hook)
- `sitemap-index.xml` + `sitemap-0.xml` — auto-generated by `@astrojs/sitemap`
- `robots.txt`, `rss.xml`

## Cloudflare Pages setup

1. In the Cloudflare dashboard, go to **Workers & Pages** → **Create** → **Pages** → **Connect to Git**.
2. Connect this repository.
3. Build settings:
   - Framework preset: **Astro**
   - Build command: `pnpm build`
   - Build output directory: `dist`
   - Root directory: `/` (default)
4. Environment variables (set in Pages → Settings → Environment variables):
   - `SITE_URL` = `https://happyendpoint.com` (or the preview URL for staging)
   - `GOOGLE_SITE_VERIFICATION` = your Google Search Console verification string (optional)
   - `BING_SITE_VERIFICATION` = your Bing Webmaster verification string (optional)
   - Analytics IDs, if used:
     - `PUBLIC_GA_MEASUREMENT_ID`
     - `PUBLIC_GTM_ID`
5. Node version: add `NODE_VERSION=22.12.0` as an environment variable so Cloudflare uses the right runtime.
6. Save and trigger the first deploy.

## Custom domain

After the first successful deploy:

1. Pages project → **Custom domains** → **Set up a custom domain**.
2. Add `happyendpoint.com` (and optionally `www.happyendpoint.com`).
3. Cloudflare auto-configures the DNS records if the domain is on Cloudflare; otherwise follow the CNAME instructions.

## Cache invalidation

Cloudflare Pages serves from the edge with default caching. When you deploy, a new deploy URL is generated and your custom domain is automatically updated. For in-flight cache purging, go to **Caching** → **Purge Everything** on the zone.

## Content updates

All site content is either in source files or in `src/content/` collections (MDX). To update:

- **Add a new blog post**: create `src/content/blog/en/<slug>.mdx` with the schema frontmatter. Redeploy.
- **Add a new API, dataset, or platform**: ideally re-run migration from the legacy data source via `pnpm migrate`, or hand-author a new MDX file under `src/content/apis/`, `src/content/datasets/`, or `src/content/platforms/`. Redeploy.
- **Update branding** (color, logo): `src/styles/themes/blue.css` for colors, `public/favicon.svg` + `public/og-default.svg` for assets, `src/config/site.config.ts` for name/description.

## Re-running migration

If the legacy Next.js site at `D:/Programming/happyendpoint-website` updates its data files, regenerate Happy Endpoint's content:

1. `cd D:/Programming/happyendpoint-website && node scripts/dump-legacy.mjs`
2. `cd D:/Programming/happyendpoint-website-new && pnpm migrate`
3. Review the diff, commit, push — Pages deploys automatically.

## Deferred / post-launch

- **Contact form backend.** The `/contact` page UI is complete but the form does not yet send mail. Pick a provider (Resend / Postmark / Cloudflare Email Routing) and wire a Pages Function at `src/pages/api/contact.ts`. Flip the client-side success-UX to a real fetch against the function.
- **Per-entity OG images.** Currently all pages share `/og-default.svg`. Consider generating per-page OG images via a Pages Function or a build-time script.
- **Pricing data on API pages.** The `pricing` slot in the API schema is currently `null` for all entries. Populate when RapidAPI pricing data is available.

## Troubleshooting

- **Build fails on Pages with "node not found"**: verify `NODE_VERSION=22.12.0` is set in env vars.
- **`pnpm migrate` fails**: requires the legacy snapshot JSON files at `scripts/legacy-snapshot/`. Re-run the legacy dump script first.
- **Search returns nothing**: Pagefind indexes during `postbuild`. Confirm `dist/pagefind/` exists after a build.
- **Redirects not applying**: check `dist/_redirects` exists after build. It's composed post-build from `public/_redirects` + generated rules; if the compose step fails, the file might be missing.
