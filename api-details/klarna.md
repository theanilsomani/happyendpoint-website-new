## 💻 Developer Notes & Best Practices

To make integrating this API as simple as possible, here is some key information on how the architecture handles your queries:

- **The `region` Parameter:** Every endpoint accepts an optional `region` query parameter. This allows you to effortlessly localize your searches.
- **Accepted values:** `austria`, `denmark`, `finland`, `france`, `germany`, `ireland`, `italy`, `netherlands`, `norway`, `spain`, `sweden`, `uk`, `usa`
- **Default behavior:** If no region is provided, the API automatically defaults to `usa`.
